NVIDIA-SMI has failed：无法与nvidia driver通信问题

发表于 2021-07-19 分类于 linux 阅读次数：
本文字数： 788 阅读时长 ≈ 1 分钟

问题描述

服务器意外重启后，使用命令nvidia-smi时，遇到以下错误：

NVIDIA-SMI has failed because it couldn't communicate with the NVIDIA driver. Make sure that the latest NVIDIA driver is installed and running.

经过查阅资料，发现应该是重启时，linux内核自动更新导致原来的驱动向上不兼容。

解决方法

命令

安装DKMS，然后使用DKMS重新安装一遍nvidia驱动。

备注

  • Dynamic Kernel Module Support可以自动维护内核外的驱动程序，在内核版本变动之后，可以自动重新生成新的模块。除了这次遇到的显卡驱动的问题，其他驱动都可能会遇到这类问题。

  • 有人说先使用nvcc -V检查cuda是否仍存在，不在的话需要先重新安装。不过在我解决问题的过程中，发现nvcc -V执行失败，报错如下，但仍然能通过上述方法解决问题，而不用重新安装cuda。
